# 🧠 AI 开发提示文档（Final）

## Markdown Block Tree Editor（Roam / Logseq 风格）

---

# 1. 项目目标（必须理解）

我要开发一个：

> **基于 Markdown 的块级树结构编辑器（Outliner + Structured Markdown Editor）**

类似：

* Roam Research
* Logseq
* Notion（部分行为）

但区别是：

👉 **这是一个“结构优先”的编辑器，而不是富文本编辑器**

---

# 2. 核心设计理念（必须遵守）

---

## 2.1 双层模型（关键）

```text
Block Tree（结构层）
  ↓
Markdown AST（内容层）
```

---

## 2.2 Block 是结构单位

* 每个 block 是树节点
* block 之间形成层级关系

---

## 2.3 Markdown 是内容语义

* block.content = Markdown 文本
* 解析为 AST（标题 / 列表 / 代码块等）

---

## 2.4 重要原则

---

### 原则 1：结构优先

* 所有操作不能破坏 Block Tree

---

### 原则 2：子树不可丢失

* children 必须始终存在或迁移

---

### 原则 3：层级变更必须显式

* 只能通过 Tab / Shift+Tab

---

### 原则 4：编辑行为分层

```text
行内编辑 → Markdown 层
结构编辑 → Block Tree 层
```

---

# 3. 数据模型（建议实现，不强制细节）

---

## 3.1 Block Tree

```ts
Block {
  id
  content: string   // 原始 markdown
  parent
  children[]
}
```

---

## 3.2 Markdown AST（每个 block 内）

建议：

* 可使用现成 parser（如 remark / markdown-it）
* 或自定义轻量 AST

---

## 3.3 行模型（运行时）

```ts
Line {
  text
  astNodeRef
}
```

👉 用于 cursor 与渲染

---

# 4. 渲染策略（允许自由发挥）

---

## 必须满足：

* 每个 block 可编辑
* 支持多行
* 能映射 cursor

---

## 推荐方向（非强制）：

```html
<div data-block-id>
  <div contenteditable>
    <div data-line="0">...</div>
    <div data-line="1">...</div>
  </div>
</div>
```

---

## 可自由发挥：

* React / Vanilla
* 是否虚拟 DOM
* 是否拆分 line DOM

👉 只要满足 cursor 精确控制即可

---

# 5. 光标模型（必须实现）

---

```ts
Cursor {
  blockId
  line
  column

```

---

## 必须支持：

* DOM → Cursor
* Cursor → DOM
* column memory（上下移动）

---

# 6. 光标移动行为（必须符合）

---

## ↑ / ↓

---

### 优先规则：

👉 **优先在 block 内移动**

---

### ↑

* 若 line > 0 → line - 1
* 若 line == 0 → 上一个 block 的最后一行

---

### ↓

* 若 line < last → line + 1
* 若在最后一行 → 下一个 block 第一行

---

---

## ← / →

---

### ←

* column > 0 → 正常
* column == 0：

  * 若 line > 0 → 上一行末尾
  * 若 line == 0 → 上一个 block

---

---

### →

* 行内移动
* 行尾 → 下一行 / 下一个 block

---

---

# 7. 编辑行为（核心）

---

# 7.1 Enter

---

## 行内

👉 split line（block 内）

---

## block 边界

👉 创建新 block

---

## Markdown 语义（建议支持）

例如：

```md
- item|
```

Enter →

```md
- item
- |
```

---

👉 这里允许 AI 自由发挥实现方式

---

# 7.2 Backspace

---

## 行内

👉 删除字符

---

## 行首（非第一行）

👉 合并到上一行

---

## block 第一行开头

👉 **mergeWithPrevious（结构操作）**

---

---

# 7.3 Delete

---

## 行内

👉 删除字符

---

## 行尾（非最后一行）

👉 合并下一行

---

## block 末尾

👉 根据结构规则处理

---

---

# 8. Block 合并（必须严格实现）

---

## Backspace at block start

👉 merge current → previous

---

## Delete at block end

---

### 仅当：

👉 next block 是同级

才允许 merge

---

---

## 合并规则

---

### 文本

```text
target.lastLine += source.firstLine
```

---

### 行

* source 剩余行追加

---

### 子节点

```text
source.children → target.children
```

---

### 删除 source

---

---

## 禁止行为

* ❌ 跨层级 merge
* ❌ 丢失 children
* ❌ 自动改变层级

---

# 9. Markdown AST 行为（关键升级）

---

## 9.1 AST 的作用

* 解析结构（列表 / 标题 / 代码块）
* 提供语义信息
* 但不直接控制 Block Tree

---

---

## 9.2 Block 与 AST 的关系

---

👉 推荐：

```text
Block = AST root 的一个容器
```

---

---

## 9.3 不要做的事（重要）

* ❌ 不要让 Markdown AST 决定 block 层级
* ❌ 不要自动把 Markdown list 转成 tree

---

👉 原因：

> Block Tree = 用户显式结构
> Markdown = 内容语义

---

---

## 9.4 可以实现的增强（可选）

---

* 自动 continuation（列表）
* 代码块 enter 行为
* 标题自动补全

👉 这些允许自由发挥

---
