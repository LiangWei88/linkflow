## 1. 项目概述

### 1.1 目标
开发一个独立的大纲笔记编辑器组件，支持树形结构、键盘导航、无感编辑体验。

### 1.2 范围
- 只负责编辑器 UI 和交互逻辑
- 不涉及数据持久化（后端 API）
- 使用 mock 数据进行开发和演示
- 预留与现有数据 API 兼容的接口

### 1.3 技术栈
- React 18+
- TypeScript
- Tailwind CSS
- 原生 contenteditable（不使用 Tiptap 等富文本编辑器）

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 树形大纲展示
- 支持无限层级嵌套
- 每个节点称为 "block"
- 展示层级缩进（视觉区分）
- 支持折叠/展开子节点

#### 2.1.2 无感编辑体验
- 点击任意 block 进入编辑模式
- 编辑和预览状态视觉完全一致（无边框、无背景变化）
- 点击哪里，光标就出现在哪里
- 同一时间只有一个 block 处于编辑模式

#### 2.1.3 键盘导航

| 按键 | 行为 |
|------|------|
| ↑ | 光标在第一行时，跳到上一个 block 的最后一行，保持水平位置 |
| ↓ | 光标在最后一行时，跳到下一个 block 的第一行，保持水平位置 |
| ← | 光标在 block 开头时，跳到上一个 block 的结尾 |
| → | 光标在 block 结尾时，跳到下一个 block 的开头 |
| Tab | 缩进（将当前 block 变为上一个 block 的子节点） |
| Shift+Tab | 取消缩进（将当前 block 提升到父节点同级） |
| Enter | 在当前位置拆分为两个 block |
| Backspace（空内容时） | 删除当前 block，光标跳到上一个 block 结尾 |
| Delete（空内容时） | 删除当前 block，光标跳到下一个 block 开头 |

**说明**：
- "上一个/下一个 block" 指扁平化遍历顺序（考虑折叠状态）
- 跨 block 导航时保持光标的水平位置（X 坐标）
- 如果按键方向还有内容，则执行默认光标移动行为

#### 2.1.4 折叠/展开
- 有子节点的 block 显示折叠按钮（箭头图标）
- 点击箭头或按 → 键展开
- 点击箭头或按 ← 键折叠
- 折叠时隐藏所有子节点

#### 2.1.5 内容编辑
- 支持多行文本（自动换行）
- 支持中文输入法
- 防抖保存（300ms）
- 空内容时显示占位符 "输入内容..."

## 3. 数据结构

### 3.1 Block 类型

```typescript
interface Block {
  id: string           // 唯一标识
  content: string      // 文本内容
  parentId: string | null  // 父节点 ID，null 表示根节点
  blockOrder: number   // 同级节点排序
  isCollapsed?: boolean // 是否折叠（可选，UI 状态）
}
```

## 4. 组件接口

### 4.1 主组件：OutlineEditor

```typescript
interface OutlineEditorProps {
  // 初始数据
  initialBlocks?: Block[]
  
  // 回调函数（预留接口，用于后续对接真实 API）
  onBlocksChange?: (blocks: Block[]) => void           // 数据变化时回调
  onBlockUpdate?: (blockId: string, content: string) => Promise<void>  // 更新内容
  onBlockCreate?: (parentId: string | null, order: number, content?: string) => Promise<Block>  // 创建 block
  onBlockDelete?: (blockId: string) => Promise<void>  // 删除 block
  onBlockMove?: (blockId: string, newParentId: string | null, newOrder: number) => Promise<void>  // 移动 block
}

export function OutlineEditor(props: OutlineEditorProps): React.ReactElement
```

### 4.2 使用示例

```tsx
// 基础用法（mock 数据）
function App() {
  return <OutlineEditor />
}

// 带初始数据
function App() {
  return <OutlineEditor initialBlocks={myBlocks} />
}

// 对接真实 API（预留接口）
function App() {
  return (
    <OutlineEditor
      initialBlocks={loadedBlocks}
      onBlockUpdate={async (id, content) => {
        await api.updateBlock(id, { content })
      }}
      onBlockCreate={async (parentId, order, content) => {
        return await api.createBlock({ parentId, blockOrder: order, content })
      }}
      onBlockDelete={async (id) => {
        await api.deleteBlock(id)
      }}
      onBlockMove={async (id, parentId, order) => {
        await api.moveBlock(id, { parentId, blockOrder: order })
      }}
    />
  )
}
```

## 7. 状态管理

### 7.1 内部状态

```typescript
interface EditorState {
  blocks: Block[]                    // 所有 block 数据
  editingBlockId: string | null      // 当前编辑的 block ID
  collapsedIds: Set<string>          // 折叠的 block ID 集合
  cursorOffset?: number              // 点击时的文本偏移量
  cursorPosition?: CursorPosition    // 导航时的目标位置
  savedCursorX?: number              // 跨 block 导航时保存的 X 坐标
}

type CursorPosition = 'start' | 'end' | 'firstLine' | 'lastLine'
```

### 7.2 状态流转

```
点击 Block
    ↓
设置 editingBlockId = block.id
设置 cursorOffset = 点击位置
    ↓
渲染 BlockEditor（编辑模式）
    ↓
初始化光标位置（使用 cursorOffset）
    ↓
用户输入
    ↓
防抖保存 → 调用 onBlockUpdate
    ↓
按方向键（边界）
    ↓
设置 cursorPosition + savedCursorX
设置 editingBlockId = 目标 block.id
    ↓
渲染 BlockEditor（编辑模式）
    ↓
初始化光标位置（使用 cursorPosition + savedCursorX）
```

## 8. 性能优化

### 8.1 渲染优化
- 只有当前编辑的 block 创建编辑器实例
- 其他 block 使用静态 div 展示
- 使用 React.memo 或 useMemo 缓存树形结构

### 8.2 输入优化
- 使用防抖（300ms）减少更新频率
- 使用 requestAnimationFrame 优化光标定位

### 8.3 内存优化
- 组件卸载时清理定时器
- 避免在渲染时创建新对象
