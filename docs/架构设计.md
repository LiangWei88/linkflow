**版本**: 1.0  
**日期**: 2026-03-21  
**状态**: MVP 阶段

---

## 1. 设计目标

基于「快速无压记录」理念，构建一款自托管、Web 化的双链大纲笔记工具。

### 核心原则

1. **快速无压**: 记录时零心智负担，整理时灵活方便
2. **自托管优先**: 数据完全由用户掌控
3. **渐进增强**: MVP 核心功能完整，预留扩展接口
4. **前后端融合**: 简化架构，降低维护成本

## 4. 关键设计决策

### 4.1 为什么前后端融合？

**决策**: 前端直接访问 SQLite，不通过 HTTP API

**理由**:
1. **简化架构**: 减少一层网络通信，降低复杂度
2. **性能优势**: 本地数据库访问延迟极低
3. **开发效率**: 无需维护 API 接口文档
4. **部署简单**: 单容器部署，无需考虑服务间通信

**限制与应对**:
- 后续如需多用户/协同，可拆分为独立后端服务
- 预留 Service 层接口，便于后续重构

### 4.2 为什么选 SQLite？

**决策**: 使用 SQLite 而非 PostgreSQL/MySQL

**理由**:
1. **零配置**: 无需安装配置数据库服务
2. **单文件**: 便于备份、迁移、版本控制
3. **自托管友好**: 用户无需维护数据库服务
4. **性能足够**: MVP 阶段数据量小，SQLite 完全满足
5. **FTS5 支持**: 内置全文搜索，无需 Elasticsearch

### 4.3 为什么选 Tiptap？

**决策**: 使用 Tiptap 而非 Slate/Quill

**理由**:
1. **扩展性强**: 基于 ProseMirror，可深度定制
2. **生态丰富**: 大量现成扩展可用
3. **TypeScript**: 原生支持，类型完善
4. **社区活跃**: 持续更新，文档完善
5. **大纲友好**: 天然支持嵌套文档结构

## 5. 数据流设计

### 5.1 笔记编辑流程

```
用户输入
   │
   ▼
Tiptap Editor ──→ 实时解析 [[wikilinks]]
   │                    │
   │                    ▼
   │            LinkService.syncBacklinks()
   │                    │
   ▼                    ▼
编辑器状态更新 ←── 反链数据更新
   │
   ▼
自动保存 (防抖 1s)
   │
   ▼
BlockService.saveBlocks()
   │
   ▼
SQLite (blocks 表)
```

### 5.2 页面加载流程

```
打开页面 /note/2026-03-21
   │
   ▼
解析路由参数 docId
   │
   ▼
DailyNoteService.getOrCreateNote(docId)
   │
   ├─→ 存在 → 返回笔记数据
   └─→ 不存在 → 创建新笔记（应用模板）
            │
            ▼
      BlockService.createNote()
            │
            ▼
      渲染编辑器
            │
            ▼
      BacklinkPanel 加载反链
```

### 5.3 搜索流程

```
用户输入关键词
   │
   ▼
防抖处理 (300ms)
   │
   ▼
SearchService.search(keyword)
   │
   ├─→ 使用 FTS5 全文索引
   │       │
   │       ▼
   │   blocks_fts 虚拟表
   │       │
   └───────┘
   │
   ▼
返回搜索结果
   │
   ▼
高亮显示匹配内容
```

## 7. 性能考虑

### 7.1 优化策略

| 场景 | 策略 | 实现 |
|------|------|------|
| 大数据量 | 虚拟滚动 | 大纲节点虚拟化 |
| 频繁保存 | 防抖 | 1s 防抖自动保存 |
| 搜索性能 | 全文索引 | FTS5 虚拟表 |
| 首屏加载 | 懒加载 | 反链面板延迟加载 |
| 内存管理 | 缓存策略 | LRU 缓存最近访问笔记 |

### 7.2 性能指标

- 首屏加载时间 < 2s
- 编辑器响应延迟 < 50ms
- 搜索响应时间 < 100ms
- 支持单文档 1000+ 大纲节点

## 8. 安全考虑

### 8.1 数据安全

- SQLite 文件权限控制（600）
- 定期自动备份（每日）
- 敏感操作确认（删除笔记）

### 8.2 输入安全

- 参数化 SQL 查询（防注入）
- XSS 防护（Tiptap 自动转义）
- 内容长度限制（单块 10MB）
