## 1. 代码风格规范

### 1.1 基础规范

#### 缩进与格式
- 使用 **2 个空格** 缩进
- 使用 **UTF-8** 编码
- 使用 **LF** 换行符（Unix 风格）
- 文件末尾保留一个空行

#### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 变量 | camelCase | `blockId`, `userName` |
| 常量 | UPPER_SNAKE_CASE | `MAX_BLOCK_SIZE`, `DEFAULT_TEMPLATE` |
| 函数 | camelCase | `getBlockById()`, `createLink()` |
| 类/接口 | PascalCase | `BlockService`, `IBacklinkService` |
| 类型别名 | PascalCase | `BlockId`, `DocContent` |
| 枚举 | PascalCase | `BlockType`, `ThemeMode` |
| 文件/目录 | kebab-case | `block-service.ts`, `backlink-panel/` |
| 组件 | PascalCase | `BacklinkPanel.tsx`, `Editor.tsx` |

## 4. Git 工作流规范

### 4.1 分支策略

```
main                    # 生产分支
├── develop             # 开发分支
├── feature/outline     # 功能分支
├── feature/wikilink
├── bugfix/fix-drag
└── hotfix/critical-fix
```

### 4.2 提交规范

使用 **Conventional Commits** 规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

#### Type 类型

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复 bug |
| `docs` | 文档更新 |
| `style` | 代码格式（不影响功能） |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具链 |

#### 示例

```bash
# 功能提交
git commit -m "feat(outline): 实现大纲折叠/展开功能"

# 修复提交
git commit -m "fix(drag): 修复拖拽时节点顺序错乱问题

- 修复了拖拽后 order 计算错误
- 添加了拖拽边界检查

Closes #123"

# 文档提交
git commit -m "docs(api): 更新 BlockService 接口文档"

# 重构提交
git commit -m "refactor(db): 优化 blocks 表查询性能

- 添加 doc_id 索引
- 使用参数化查询"
```

### 4.3 提交信息规范

```
# ✅ 正确
feat(editor): 添加自动保存功能
fix(link): 修复反链计数不准确问题
docs: 更新 README 安装说明

# ❌ 错误
update code                    # 太笼统
fix bug                        # 没有说明什么 bug
添加功能                       # 没有英文 type
feat:fix something             # 格式错误
```

## 8. 错误处理规范

### 8.1 错误类型

```typescript
// ✅ 正确：使用自定义错误类型
import { NotFoundError, ValidationError } from '@/errors';

async function getBlock(id: string): Promise<Block> {
  const block = await repository.findById(id);
  if (!block) {
    throw new NotFoundError('Block', id);
  }
  return block;
}

// ✅ 正确：统一的错误处理
async function handleOperation(): Promise<void> {
  try {
    await riskyOperation();
  } catch (error) {
    if (error instanceof AppError) {
      // 已知错误，显示友好提示
      showToast(error.message);
    } else {
      // 未知错误，记录日志
      console.error('Unexpected error:', error);
      showToast('操作失败，请重试');
    }
  }
}
```

### 8.2 错误边界

```typescript
// components/ErrorBoundary.tsx
export class ErrorBoundary extends React.Component<
  { children: React.ReactNode },
  { hasError: boolean }
> {
  static getDerivedStateFromError(): { hasError: true } {
    return { hasError: true };
  }

  componentDidCatch(error: Error, errorInfo: React.ErrorInfo): void {
    console.error('Error caught by boundary:', error, errorInfo);
    // 可发送到错误监控服务
  }

  render(): React.ReactNode {
    if (this.state.hasError) {
      return <ErrorFallback />;
    }
    return this.props.children;
  }
}
```

## 9. 性能规范

### 9.1 渲染优化

```typescript
// ✅ 正确：使用 React.memo 避免不必要的重渲染
export const BlockItem = React.memo(function BlockItem({ block, onUpdate }: Props) {
  // ...
}, (prevProps, nextProps) => {
  return prevProps.block.id === nextProps.block.id &&
         prevProps.block.content === nextProps.block.content;
});

// ✅ 正确：使用 useMemo 缓存计算
const sortedBlocks = useMemo(() => {
  return blocks.sort((a, b) => a.order - b.order);
}, [blocks]);

// ✅ 正确：使用 useCallback 缓存回调
const handleUpdate = useCallback((content: string) => {
  onUpdate(block.id, content);
}, [block.id, onUpdate]);
```

### 9.2 数据库优化

```typescript
// ✅ 正确：使用参数化查询
const stmt = db.prepare('SELECT * FROM blocks WHERE doc_id = ?');
const blocks = stmt.all(docId);

// ✅ 正确：批量操作使用事务
db.exec('BEGIN TRANSACTION');
try {
  for (const block of blocks) {
    insertStmt.run(block.id, block.content);
  }
  db.exec('COMMIT');
} catch (err) {
  db.exec('ROLLBACK');
  throw err;
}

// ✅ 正确：限制查询结果数量
```
