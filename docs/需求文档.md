# 需求文档

**版本**: 1.0  
**日期**: 2026-03-21  
**状态**: MVP 阶段

---

## 1. 产品概述

### 1.1 产品定位

一款**自托管、Web 化、大纲 + 双向链接**的笔记工具，支持**快速记录 + 高效整理**。

### 1.2 核心理念

基于「快速无压记录」理念，让用户在记录时零心智负担，整理时灵活方便。

### 1.3 目标用户

- 个人知识管理者
- 需要自托管数据的用户
- 追求轻量级笔记工具的用户

---

## 2. 功能需求

### 2.1 大纲编辑器

**优先级**: ★★★★★

#### 功能描述

| 功能 | 说明 |
|------|------|
| 无限层级 | 支持多级嵌套大纲（如 `1 → 1.1 → 1.1.1`） |
| 折叠/展开 | 每个节点可独立折叠或展开 |
| 拖拽排序 | 支持鼠标/触控拖拽调整顺序，拖动到另一条上面时变成该条的下级内容 |
| 所见即所得 | 实时预览 Markdown 渲染效果 |
| 点击编辑 | 点击节点可直接修改内容 |
| Markdown | 兼容基础 Markdown 语法 |

#### 验收标准

- [ ] 创建多级大纲，验证折叠/展开功能正常
- [ ] 拖拽节点后顺序正确更新
- [ ] 拖动节点到另一条上面时正确变成下级内容
- [ ] 点击节点可直接进入编辑模式
- [ ] `**粗体**`、`- 列表`、`[链接](url)` 正确渲染
- [ ] DOM 结构符合预期（`<li data-type="outline_item">`）

#### 技术要点

- 使用 Tiptap 扩展 `Node` 实现大纲节点
- 集成 `@dnd-kit` 实现拖拽功能，支持层级调整
- 实现点击编辑的交互逻辑
- 优化拖拽时的视觉反馈

---

### 2.2 双向链接

**优先级**: ★★★★★

#### 功能描述

| 功能 | 说明 |
|------|------|
| Wikilink 语法 | 使用 `[[链接名]]` 创建链接 |
| 自动补全 | 输入 `[[` 时弹出已有块列表 |
| 跳转导航 | 点击链接跳转到目标块 |
| 反链面板 | 展示当前块的反向链接，支持拖拽到正文 |

#### 验收标准

- [ ] 输入 `[[` 时弹出补全菜单
- [ ] 点击 `[[链接]]` 可跳转到目标块
- [ ] 反链数据正确存储到 SQLite
- [ ] 反链面板展示无误，支持拖拽

#### 技术要点

- 扩展 Tiptap 的 `Mark` 实现 `[[wikilinks]]`
- 使用 `backlinks` 表存储链接关系

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 首屏加载时间 | < 2s |
| 编辑器响应延迟 | < 50ms |
| 搜索响应时间 | < 100ms |
| 单文档支持节点数 | 1000+ |

### 3.2 安全要求

| 要求 | 说明 |
|------|------|
| 数据安全 | SQLite 文件权限控制 |
| 输入安全 | 参数化 SQL 查询，防注入 |
| XSS 防护 | Tiptap 自动转义用户输入 |
| 定期备份 | 每日自动备份数据 |

### 3.3 兼容性要求

| Firefox | 最新 2 个版本 |

| iOS Safari | iOS 14+ |

## 5. 风险与规避

| 风险 | 可能性 | 影响 | 规避措施 |
|------|--------|------|----------|
| SQLite 性能瓶颈 | 中 | 高 | 添加索引和分页查询，监控性能 |
| Tiptap 复杂度高 | 中 | 中 | 使用现成扩展，逐步定制 |
| 移动端适配困难 | 高 | 中 | 使用 PWA + 触控优化库 |
| 数据损坏 | 低 | 高 | 自动备份 + 数据校验 |
