# 数据库设计文档

**版本**: 1.0  
**日期**: 2026-03-21  
**数据库**: SQLite 3.x

---

## 1. 概述

本文档定义双链大纲笔记软件的数据库结构，包括表设计、索引、关系和约束。

### 设计原则

1. **简洁性**: 最小化表数量，避免过度设计
2. **扩展性**: 预留字段和表，支持未来功能
3. **性能**: 合理设计索引，支持 FTS5 全文搜索
4. **一致性**: 统一命名规范，外键约束

## 3. 表结构

### 3.1 blocks - 块表

存储所有笔记块（大纲节点）的内容和层级关系。

```sql
CREATE TABLE blocks (
  id TEXT PRIMARY KEY,              -- 块唯一标识 (UUID)
  doc_id TEXT NOT NULL,             -- 文档ID (如 "2026-03-21" 或自定义页面名)
  content TEXT NOT NULL DEFAULT '', -- 块内容 (Markdown/HTML)
  parent_id TEXT,                   -- 父块ID (自引用，支持无限层级)
  "order" INTEGER DEFAULT 0,        -- 同级排序 (注意: order 是 SQL 关键字，需引号)
  collapsed INTEGER DEFAULT 0,      -- 是否折叠 (0=展开, 1=折叠)
  created_at INTEGER NOT NULL,      -- 创建时间戳 (Unix timestamp)
  updated_at INTEGER NOT NULL,      -- 更新时间戳 (Unix timestamp)
  
  -- 外键约束
  FOREIGN KEY (parent_id) REFERENCES blocks(id) ON DELETE CASCADE
);
```

**字段说明**:

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PK | UUID v4，全局唯一 |
| doc_id | TEXT | NOT NULL | 文档标识，Daily Notes 使用日期格式 |
| content | TEXT | NOT NULL | 支持 Markdown 和 HTML |
| parent_id | TEXT | FK | 自引用，null 表示根节点 |
| order | INTEGER | DEFAULT 0 | 同级节点排序，越小越靠前 |
| collapsed | INTEGER | DEFAULT 0 | UI 状态，是否折叠子节点 |
| created_at | INTEGER | NOT NULL | Unix 时间戳（秒） |
| updated_at | INTEGER | NOT NULL | Unix 时间戳（秒） |

**示例数据**:

```sql
-- Daily Note 结构示例
INSERT INTO blocks VALUES 
  ('uuid-1', '2026-03-21', '# 今日笔记', NULL, 0, 0, 1711000000, 1711000000),
  ('uuid-2', '2026-03-21', '- [[项目A]] 进展', 'uuid-1', 0, 0, 1711000100, 1711000100),
  ('uuid-3', '2026-03-21', '  - 完成了登录功能', 'uuid-2', 0, 0, 1711000200, 1711000200);
```

### 3.2 backlinks - 反链表

存储双向链接关系。

```sql
CREATE TABLE backlinks (
  block_id TEXT NOT NULL,           -- 源块ID (引用者)
  target_id TEXT NOT NULL,          -- 目标块ID (被引用者)
  created_at INTEGER DEFAULT (strftime('%s', 'now')), -- 创建时间
  
  -- 复合主键
  PRIMARY KEY (block_id, target_id),
  
  -- 外键约束
  FOREIGN KEY (block_id) REFERENCES blocks(id) ON DELETE CASCADE,
  FOREIGN KEY (target_id) REFERENCES blocks(id) ON DELETE CASCADE
);
```

**字段说明**:

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| block_id | TEXT | PK, FK | 源块ID，包含 [[link]] 的块 |
| target_id | TEXT | PK, FK | 目标块ID，被引用的块 |
| created_at | INTEGER | DEFAULT | 链接创建时间 |

**示例数据**:

```sql
-- block-1 引用了 block-2
INSERT INTO backlinks (block_id, target_id) VALUES 
  ('block-1', 'block-2');
```

### 3.3 blocks_fts - 全文搜索虚拟表

FTS5 全文索引，支持内容搜索。

```sql
-- 创建 FTS5 虚拟表
CREATE VIRTUAL TABLE blocks_fts USING fts5(
  content,                          -- 索引内容
  content='blocks',                 -- 数据来源表
  content_rowid='id'                -- 关联字段
);

-- 创建触发器，自动同步索引
CREATE TRIGGER blocks_fts_insert AFTER INSERT ON blocks BEGIN
  INSERT INTO blocks_fts(rowid, content) VALUES (new.id, new.content);
END;

CREATE TRIGGER blocks_fts_delete AFTER DELETE ON blocks BEGIN
  INSERT INTO blocks_fts(blocks_fts, rowid, content) VALUES ('delete', old.id, old.content);
END;

CREATE TRIGGER blocks_fts_update AFTER UPDATE ON blocks BEGIN
  INSERT INTO blocks_fts(blocks_fts, rowid, content) VALUES ('delete', old.id, old.content);
  INSERT INTO blocks_fts(rowid, content) VALUES (new.id, new.content);
END;
```

**使用示例**:

```sql
-- 全文搜索
SELECT b.* FROM blocks b
JOIN blocks_fts fts ON b.id = fts.rowid
WHERE fts.content MATCH '项目'
ORDER BY rank;

-- 前缀搜索
SELECT b.* FROM blocks b
JOIN blocks_fts fts ON b.id = fts.rowid
WHERE fts.content MATCH '项目*'
LIMIT 20;
```

### 3.4 attachments - 附件表

存储附件文件元数据。

```sql
CREATE TABLE attachments (
  id TEXT PRIMARY KEY,              -- 附件ID (UUID)
  block_id TEXT,                    -- 关联块ID (可为空)
  filename TEXT NOT NULL,           -- 原始文件名
  storage_name TEXT NOT NULL,       -- 存储文件名 (UUID + 扩展名)
  mime_type TEXT,                   -- MIME 类型
  size INTEGER,                     -- 文件大小 (字节)
  created_at INTEGER DEFAULT (strftime('%s', 'now'))
);
```

**字段说明**:

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PK | UUID |
| block_id | TEXT | FK | 关联块，null 表示未关联 |
| filename | TEXT | NOT NULL | 用户上传时的文件名 |
| storage_name | TEXT | NOT NULL | 实际存储文件名 |
| mime_type | TEXT | - | 如 image/png |
| size | INTEGER | - | 文件大小 |
| created_at | INTEGER | DEFAULT | 上传时间 |

### 3.5 settings - 设置表

存储用户配置和系统状态。

```sql
CREATE TABLE settings (
  key TEXT PRIMARY KEY,             -- 设置键
  value TEXT,                       -- 设置值 (JSON 字符串)
  updated_at INTEGER DEFAULT (strftime('%s', 'now'))
);
```

**预设配置项**:

| key | value | 说明 |
|-----|-------|------|
| daily_note_template | `{"content": "# {{date}}\n- "}` | Daily Note 模板 |
| editor_font_size | `16` | 编辑器字体大小 |
| theme | `"light"` | 主题设置 |
| last_opened_doc | `"2026-03-21"` | 最后打开的文档 |
| version | `"1.0.0"` | 数据库版本 |

## 4. 索引设计

### 4.1 已自动创建的索引

- `blocks.id` - PRIMARY KEY 自动创建
- `backlinks(block_id, target_id)` - PRIMARY KEY 自动创建

### 4.2 需要手动创建的索引

```sql
-- blocks 表索引
CREATE INDEX idx_blocks_doc_id ON blocks(doc_id);           -- 按文档查询
CREATE INDEX idx_blocks_parent_id ON blocks(parent_id);     -- 查询子节点
CREATE INDEX idx_blocks_updated ON blocks(updated_at DESC); -- 最近更新

-- backlinks 表索引
CREATE INDEX idx_backlinks_target ON backlinks(target_id);  -- 反链查询

-- attachments 表索引
CREATE INDEX idx_attachments_block ON attachments(block_id);
```

### 4.3 索引使用场景

| 查询场景 | 使用索引 | 说明 |
|---------|---------|------|
| 获取文档所有块 | idx_blocks_doc_id | Daily Note 加载 |
| 获取子节点 | idx_blocks_parent_id | 大纲渲染 |
| 获取反链 | idx_backlinks_target | 反链面板 |
| 最近更新 | idx_blocks_updated | 首页列表 |

## 5. 视图（可选）

### 5.1 文档列表视图

```sql
CREATE VIEW v_docs AS
SELECT 
  doc_id,
  COUNT(*) as block_count,
  MAX(updated_at) as last_updated
FROM blocks
WHERE parent_id IS NULL  -- 只统计根节点
GROUP BY doc_id
ORDER BY last_updated DESC;
```

### 5.2 反链详情视图

```sql
CREATE VIEW v_backlinks_detail AS
SELECT 
  bl.target_id,
  bl.block_id as source_id,
  b.content as source_content,
  b.doc_id as source_doc,
  bl.created_at
FROM backlinks bl
JOIN blocks b ON bl.block_id = b.id;
```

## 6. 数据库初始化脚本

```sql
-- init.sql
-- 执行顺序很重要！

-- 1. 创建 blocks 表
CREATE TABLE IF NOT EXISTS blocks (
  id TEXT PRIMARY KEY,
  doc_id TEXT NOT NULL,
  content TEXT NOT NULL DEFAULT '',
  parent_id TEXT,
  "order" INTEGER DEFAULT 0,
  collapsed INTEGER DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  FOREIGN KEY (parent_id) REFERENCES blocks(id) ON DELETE CASCADE
);

-- 2. 创建 backlinks 表
CREATE TABLE IF NOT EXISTS backlinks (
  block_id TEXT NOT NULL,
  target_id TEXT NOT NULL,
  created_at INTEGER DEFAULT (strftime('%s', 'now')),
  PRIMARY KEY (block_id, target_id),
  FOREIGN KEY (block_id) REFERENCES blocks(id) ON DELETE CASCADE,
  FOREIGN KEY (target_id) REFERENCES blocks(id) ON DELETE CASCADE
);

-- 3. 创建 FTS5 虚拟表
CREATE VIRTUAL TABLE IF NOT EXISTS blocks_fts USING fts5(
  content,
  content='blocks',
  content_rowid='id'
);

-- 4. 创建 FTS5 触发器
CREATE TRIGGER IF NOT EXISTS blocks_fts_insert AFTER INSERT ON blocks BEGIN
  INSERT INTO blocks_fts(rowid, content) VALUES (new.id, new.content);
END;

CREATE TRIGGER IF NOT EXISTS blocks_fts_delete AFTER DELETE ON blocks BEGIN
  INSERT INTO blocks_fts(blocks_fts, rowid, content) VALUES ('delete', old.id, old.content);
END;

CREATE TRIGGER IF NOT EXISTS blocks_fts_update AFTER UPDATE ON blocks BEGIN
  INSERT INTO blocks_fts(blocks_fts, rowid, content) VALUES ('delete', old.id, old.content);
  INSERT INTO blocks_fts(rowid, content) VALUES (new.id, new.content);
END;

-- 5. 创建 attachments 表
CREATE TABLE IF NOT EXISTS attachments (
  id TEXT PRIMARY KEY,
  block_id TEXT,
  filename TEXT NOT NULL,
  storage_name TEXT NOT NULL,
  mime_type TEXT,
  size INTEGER,
  created_at INTEGER DEFAULT (strftime('%s', 'now'))
);

-- 6. 创建 settings 表
CREATE TABLE IF NOT EXISTS settings (
  key TEXT PRIMARY KEY,
  value TEXT,
  updated_at INTEGER DEFAULT (strftime('%s', 'now'))
);

-- 7. 创建索引
CREATE INDEX IF NOT EXISTS idx_blocks_doc_id ON blocks(doc_id);
CREATE INDEX IF NOT EXISTS idx_blocks_parent_id ON blocks(parent_id);
CREATE INDEX IF NOT EXISTS idx_blocks_updated ON blocks(updated_at DESC);
CREATE INDEX IF NOT EXISTS idx_backlinks_target ON backlinks(target_id);
CREATE INDEX IF NOT EXISTS idx_attachments_block ON attachments(block_id);

-- 8. 插入默认设置
INSERT OR IGNORE INTO settings (key, value) VALUES 
  ('version', '1.0.0'),
  ('daily_note_template', '{"content": "# {{date}}\\n- "}'),
  ('theme', 'light'),
  ('editor_font_size', '16');
```

## 7. 迁移策略

### 7.1 版本控制

使用 `settings` 表记录数据库版本：

```sql
-- 检查当前版本
SELECT value FROM settings WHERE key = 'version';

-- 更新版本
UPDATE settings SET value = '1.1.0' WHERE key = 'version';
```

### 7.2 迁移脚本示例

```sql
-- migration_1.0.0_to_1.1.0.sql
-- 添加新字段示例

ALTER TABLE blocks ADD COLUMN color TEXT;

UPDATE settings SET value = '1.1.0' WHERE key = 'version';
```

## 8. 性能优化建议

### 8.1 查询优化

```sql
-- ✅ 使用索引
SELECT * FROM blocks WHERE doc_id = ?;

-- ✅ 限制结果数量
SELECT * FROM blocks WHERE doc_id = ? LIMIT 100;

-- ✅ 使用 FTS5 全文搜索
SELECT b.* FROM blocks b
JOIN blocks_fts fts ON b.id = fts.rowid
WHERE fts.content MATCH ?;

-- ❌ 避免全表扫描
SELECT * FROM blocks WHERE content LIKE '%keyword%';

-- ❌ 避免大数据量查询
SELECT * FROM blocks;  -- 没有 WHERE 和 LIMIT
```

### 8.2 写入优化

```sql
-- ✅ 使用事务批量写入
BEGIN TRANSACTION;
INSERT INTO blocks VALUES (...);
INSERT INTO blocks VALUES (...);
COMMIT;

-- ✅ 使用参数化查询
const stmt = db.prepare('INSERT INTO blocks VALUES (?, ?, ?)');
stmt.run(id1, docId1, content1);
stmt.run(id2, docId2, content2);
```

## 附录 A: 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 表名 | 小写，复数 | `blocks`, `backlinks` |
| 字段 | 小写，下划线分隔 | `created_at`, `parent_id` |
| 主键 | `id` | `id` |
| 外键 | `[表名]_id` | `block_id`, `parent_id` |
| 时间戳 | `[动作]_at` | `created_at`, `updated_at` |
| 布尔值 | 整数 (0/1) | `collapsed` |

## 附录 B: 类型映射

| SQLite 类型 | TypeScript 类型 | 说明 |
|-------------|-----------------|------|
| TEXT | string | UUID, 内容, 日期字符串 |
| INTEGER | number | 时间戳, 计数, 布尔 |
| REAL | number | 浮点数 (本项目中少用) |
