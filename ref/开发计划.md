在 **MVP 阶段**，结合您的**双链大纲笔记软件**需求（Tiptap + SQLite + TypeScript + 自托管），以下是从**架构师+技术专家**角度出发的**分层设计建议**和**细化开发步骤**，旨在平衡**开发效率**、**可维护性**和**扩展性**。

---

## 一、 是否需要分层设计？
**结论**：**MVP阶段采用轻量级分层，避免过度设计，但保留扩展性**。
**原因**：
1. **团队规模**：您作为**产品经理+技术审核+老板**，开发资源有限，过度分层会增加开发和维护成本。
2. **项目规模**：MVP阶段核心功能（大纲+双向链接+自托管）不需要复杂的微服务或严格分层。
3. **技术栈**：TypeScript + SQLite + Tiptap **天然支持全栈开发**，前后端融合可简化部署和调试。

**建议分层**：
| 层次       | 责任                     | 实现方式                          | 备注                                  |
|------------|--------------------------|-----------------------------------|---------------------------------------|
| **UI层**   | 界面渲染、交互           | React + Tiptap                   | 纯前端，无需API调用                  |
| **应用层** | 业务逻辑、数据处理       | TypeScript（共享代码）            | 前后端共用，如`services/block.ts`   |
| **数据层** | 数据存储、查询           | SQLite（直接集成）               | 无需独立API，前端直接调用SQLite       |
| **存储层** | 文件/数据库              | SQLite文件 + 附件存储             | 使用`better-sqlite3`或`TypeORM`       |

**关键点**：
- **前后端融合**：使用**Next.js**或**Electron**（如需桌面端），前端直接调用SQLite（无需HTTP API）。
- **共享代码**：业务逻辑（如双向链接解析）放在`/services`，前后端共用。
- **扩展性**：保留接口（如`BlockService`），后续可拆分为独立API。

## 二、 细化开发步骤（分层视角）

### **步骤1：项目骨架与数据层**
**目标**：搭建项目结构，实现SQLite数据层。
**分层实现**：
1. **存储层**：
   - 初始化SQLite数据库，定义`blocks`和`backlinks`表。
   - 示例：
     ```typescript
     // db/schema.ts
     export const initDB = () => {
       const db = new Database('notes.db');
       db.exec(`
         CREATE TABLE IF NOT EXISTS blocks (...);
         CREATE TABLE IF NOT EXISTS backlinks (...);
       `);
       return db;
     };
     ```
2. **数据层**：
   - 封装SQLite操作（如`BlockRepository`）：
     ```typescript
     // repositories/block.ts
     export class BlockRepository {
       constructor(private db: Database) {}
       async findByDoc(docId: string) {
         return this.db.prepare('SELECT * FROM blocks WHERE doc_id = ?').all(docId);
       }
     }
     ```
3. **应用层**：
   - 定义`BlockService`，整合数据层逻辑：
     ```typescript
     // services/block.ts
     export class BlockService {
       constructor(private repo: BlockRepository) {}
       async getBlocks(docId: string) {
         return this.repo.findByDoc(docId);
       }
     }
     ```

**验收标准**：
- [ ] SQLite数据库初始化成功，表结构正确。
- [ ] `BlockRepository`和`BlockService`可单元测试。

---

### **步骤2：UI层集成Tiptap**
**目标**：集成Tiptap，支持基础Markdown。
**分层实现**：
1. **UI层**：
   - 创建Tiptap编辑器组件：
     ```tsx
     // components/Editor.tsx
     import { useEditor } from '@tiptap/react';
     export function Editor({ docId }: { docId: string }) {
       const editor = useEditor({ extensions: [StarterKit] });
       return <EditorContent editor={editor} />;
     }
     ```
2. **应用层**：
   - 连接`BlockService`加载笔记内容：
     ```typescript
     // hooks/useBlocks.ts
     export function useBlocks(docId: string) {
       const [blocks, setBlocks] = useState([]);
       useEffect(() => {
         new BlockService(new BlockRepository(db)).getBlocks(docId).then(setBlocks);
       }, [docId]);
       return blocks;
     }
     ```

**验收标准**：
- [ ] Tiptap编辑器渲染成功，可输入Markdown。
- [ ] 笔记内容从SQLite加载并显示。

---

### **步骤3：实现大纲功能**
**目标**：支持无限层级大纲，折叠/展开。
**分层实现**：
1. **数据层**：
   - 扩展`blocks`表，添加`parent_id`和`order`字段。
2. **应用层**：
   - 添加`OutlineService`处理大纲层级：
     ```typescript
     // services/outline.ts
     export class OutlineService {
       async getChildren(parentId: string) {
         return db.prepare('SELECT * FROM blocks WHERE parent_id = ? ORDER BY "order"').all(parentId);
       }
     }
     ```
3. **UI层**：
   - 扩展Tiptap Schema，支持嵌套列表：
     ```typescript
     // extensions/outline.ts
     import { Node } from '@tiptap/core';
     export const OutlineItem = Node.create({
       name: 'outline_item',
       content: 'inline*',
       defining: true,
     });
     ```

**验收标准**：
- [ ] 可创建多级大纲，折叠/展开功能正常。
- [ ] 数据库正确存储层级关系。

### **步骤4：双向链接与反链**
**目标**：支持`[[wikilinks]]`，自动补全，反链展示。
**分层实现**：
1. **数据层**：
   - 扩展`backlinks`表，存储链接关系。
2. **应用层**：
   - 添加`LinkService`处理链接解析和反链查询：
     ```typescript
     // services/link.ts
     export class LinkService {
       async createLink(sourceId: string, targetId: string) {
         await db.prepare('INSERT INTO backlinks VALUES (?, ?)').run(sourceId, targetId);
       }
       async getBacklinks(targetId: string) {
         return db.prepare('SELECT * FROM backlinks WHERE target_id = ?').all(targetId);
       }
     }
     ```
3. **UI层**：
   - 集成`tiptap-extension-wikilink`，实现补全和跳转：
     ```typescript
     // extensions/wikilink.ts
     import { Wikilink } from 'tiptap-extension-wikilink';
     const editor = useEditor({
       extensions: [Wikilink.configure({ linkService: new LinkService(db) })],
     });
     ```

**验收标准**：
- [ ] 输入`[[`时弹出补全菜单。
- [ ] 反链数据正确存储和展示。

### **步骤5：Daily Notes与模板**
**目标**：每日自动创建笔记，支持模板。
**分层实现**：
1. **应用层**：
   - 添加`DailyNoteService`：
     ```typescript
     // services/daily-note.ts
     export class DailyNoteService {
       async createIfNotExists(date: string) {
         const exists = await db.prepare('SELECT 1 FROM blocks WHERE doc_id = ?').get(date);
         if (!exists) {
           await db.prepare('INSERT INTO blocks VALUES (?, ?, ?)').run(
             uuidv4(), date, '# 今日笔记\n- [[任务1]]'
           );
         }
       }
     }
     ```
2. **UI层**：
   - 启动时自动检查并创建Daily Note：
     ```typescript
     // App.tsx
     useEffect(() => {
       new DailyNoteService(db).createIfNotExists(format(new Date(), 'yyyy-MM-dd'));
     }, []);
     ```

**验收标准**：
- [ ] 每日启动时自动创建笔记。
- [ ] 模板内容正确加载。

### **步骤7：搜索与导航**
**目标**：支持全局搜索，快速跳转。
**分层实现**：
1. **数据层**：
   - 创建FTS5全文索引：
     ```sql
     CREATE VIRTUAL TABLE blocks_fts USING fts5(content);
     ```
2. **应用层**：
   - 添加`SearchService`：
     ```typescript
     // services/search.ts
     export class SearchService {
       async search(keyword: string) {
         return db.prepare('SELECT * FROM blocks_fts WHERE content MATCH ?').all(keyword);
       }
     }
     ```
3. **UI层**：
   - 集成`tiptap-extension-command-palette`：
     ```typescript
     // components/SearchPanel.tsx
     const results = await new SearchService(db).search(keyword);
     ```

**验收标准**：
- [ ] 搜索结果实时展示，高亮匹配。
- [ ] `Ctrl+P`可快速跳转。

## 三、 技术栈与工具链
| 类别       | 技术选型               | 备注                                  |
|------------|------------------------|---------------------------------------|
| **前端**   | React + Tiptap         | 纯UI层，无需API调用                  |
| **后端**   | Node.js（Express）     | 仅用于静态文件服务，无复杂逻辑        |
| **数据库** | SQLite                | 前端直接访问，无需API中转            |
| **部署**   | Docker                 | 一键部署，数据持久化                |
| **协同**   | Yjs（后续扩展）        | MVP阶段不实现，预留接口              |
| **搜索**   | SQLite FTS5            | 内嵌全文索引，无需Elasticsearch       |

| 决策点               | 方案                          | 风险规避                          |
