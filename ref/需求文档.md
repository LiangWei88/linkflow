# **双链大纲笔记软件 MVP 需求文档**
**版本**：1.0
**日期**：2026-03-21
**目标**：实现一款**自托管、Web化、大纲+双向链接**的笔记工具，支持**快速记录+高效整理**。

---

## 一、 核心功能需求

### 1. **大纲编辑器**
**描述**：
- 支持**无限层级嵌套**的大纲节点（如`1 → 1.1 → 1.1.1`）。
- 每个节点可**折叠/展开**。
- 支持**拖拽排序**（鼠标/触控）。
- 兼容**Markdown**语法（如`**粗体**`、`- 列表`、`[链接](url)`）。

**验收标准**：
- [ ] 创建多级大纲，验证折叠/展开和拖拽功能。
- [ ] 支持基础Markdown渲染（如`**粗体**`、`- 列表`）。
- [ ] DOM结构符合预期（如`<li data-type="outline_item">`）。

**技术实现**：
- 使用**Tiptap**扩展`Node`实现大纲节点：
  ```typescript
  // extensions/outline.ts
  import { Node } from '@tiptap/core';
  export const OutlineItem = Node.create({
    name: 'outline_item',
    content: 'inline*',
    parseHTML: () => [{ tag: 'li[data-type="outline_item"]' }],
    renderHTML: ({ HTMLAttributes }) => ['li', HTMLAttributes, 0],
  });
  ```
- 集成`react-dnd`实现拖拽。

---

### 2. **双向链接**
**描述**：
- 使用`[[wikilinks]]`语法创建链接（如`[[主题]]`）。
- 支持**自动补全**：输入`[[`时弹出已有块列表。
- 点击链接可**跳转到目标块**。
- **反链面板**：展示当前块的反向链接，支持拖拽到正文。

**验收标准**：
- [ ] 输入`[[`时弹出补全菜单。
- [ ] 点击`[[链接]]`可跳转到目标块。
- [ ] 反链数据正确存储到SQLite，面板展示无误。

**技术实现**：
- 扩展Tiptap的`Mark`实现`[[wikilinks]]`：
  ```typescript
  // extensions/wikilink.ts
  import { Mark } from '@tiptap/core';
  export const Wikilink = Mark.create({
    name: 'wikilink',
    parseHTML: () => [{ tag: 'a[href^="block:"]' }],
    renderHTML: ({ mark }) => ['a', { href: `block:${mark.attrs.id}` }, 0],
  });
  ```
- 反链存储：
  ```sql
  CREATE TABLE backlinks (
    block_id TEXT,
    target_id TEXT,
    PRIMARY KEY(block_id, target_id)
  );
  ```

### 3. **Daily Notes**
**描述**：
- 每日**自动创建**笔记（如`2026-03-21`）。
- 支持**自定义模板**（如“今日任务”、“灵感”）。
- 快速记录：
  - 快捷键（如`Ctrl+K`）插入`[[ ]]`。
  - 支持**图片/附件**拖拽插入。

**验收标准**：
- [ ] 每日启动时自动创建当日笔记。
- [ ] 模板内容正确加载。
- [ ] 快捷键可快速插入双向链接。

**技术实现**：
- 自动生成逻辑：
  ```typescript
  // utils/daily-notes.ts
  const today = format(new Date(), 'yyyy-MM-dd');
  const exists = db.prepare('SELECT 1 FROM blocks WHERE doc_id = ?').get(today);
  if (!exists) {
    db.prepare('INSERT INTO blocks VALUES (?, ?, ?)').run(
      uuidv4(), today, '# 今日笔记\n- [[任务1]]'
    );
  }
  ```

### 4. **自托管与Web化**
**描述**：
- 支持**Docker一键部署**。
- **Web端访问**：浏览器直接使用，无需安装客户端。
- **离线模式**：本地缓存数据，网络恢复后自动同步。

**验收标准**：
- [ ] 可通过`docker-compose up`启动服务。
- [ ] Web端访问正常，数据持久化。

**技术实现**：
- Docker配置：
  ```yaml
  # docker-compose.yml
  version: '3'
  services:
    app:
      build: .
      ports:
        - "3000:3000"
      volumes:
        - ./data:/app/data  # 持久化SQLite数据
  ```

### 5. **基础搜索与导航**
**描述**：
- **全局搜索**：支持模糊搜索、标签筛选。
- **快速导航**：`Ctrl+P`弹出页面跳转面板。

**验收标准**：
- [ ] 搜索结果实时展示，高亮匹配。
- [ ] `Ctrl+P`可快速跳转。

**技术实现**：
- 全文索引：
  ```sql
  CREATE VIRTUAL TABLE blocks_fts USING fts5(content);
  ```
- 查询：
  ```typescript
  const results = db.prepare('SELECT * FROM blocks_fts WHERE content MATCH ?').all(keyword);
  ```

## 二、 技术架构

### 1. **分层设计**
| 层次       | 责任                     | 实现方式                          |
|------------|--------------------------|-----------------------------------|
| **UI层**   | 界面渲染、交互           | React + Tiptap                   |
| **应用层** | 业务逻辑、数据处理       | TypeScript（共享代码）            |
| **数据层** | 数据存储、查询           | SQLite（直接集成）               |
| **存储层** | 文件/数据库              | SQLite文件 + 附件存储             |

### 2. **技术选型**
| 组件          | 技术选型               | 备注                                  |
|---------------|------------------------|---------------------------------------|
| 前端          | React + Tiptap         | 纯UI层，无需API调用                  |
| 后端          | Node.js（Express）     | 仅用于静态文件服务，无复杂逻辑        |
| 数据库        | SQLite                | 前端直接访问，无需API中转            |
| 部署          | Docker                 | 一键部署，数据持久化                |
| 协同          | Yjs（后续扩展）        | MVP阶段不实现，预留接口              |
| 搜索          | SQLite FTS5            | 内嵌全文索引，无需Elasticsearch       |

## 六、 附录
### 1. **数据库Schema**
```sql
CREATE TABLE blocks (
  id TEXT PRIMARY KEY,
  doc_id TEXT,
  content TEXT,
  parent_id TEXT,
  order INTEGER,
  created_at INTEGER,
  updated_at INTEGER,
  FOREIGN KEY(parent_id) REFERENCES blocks(id)
);

CREATE TABLE backlinks (
  block_id TEXT,
  target_id TEXT,
  PRIMARY KEY(block_id, target_id)
);

CREATE VIRTUAL TABLE blocks_fts USING fts5(content);
```
